# Привязка артикулов к цифрам на PDF

Локальное desktop-приложение для Windows. Оно открывает PDF или изображение со схемой, распознает цифры на странице, связывает найденные номера с артикулами из вставленного списка и экспортирует результат в CSV или JSON.

## Полуавтоматический режим v4.19

1. Нажмите `Добавить файлы` и выберите сразу несколько PDF или изображений.
2. Переключайтесь между деталировками в очереди слева или стрелками.
3. Для каждого файла вставьте собственный список номеров, артикулов и названий.
4. Создавайте обрезки и фрагменты обычным способом.
5. Запускайте поиск только после подготовки текущего файла.

При переключении в памяти сохраняются список, текущая страница PDF, фрагменты, точки и ручные исправления. Статус рядом с файлом показывает, нужен ли список, готов ли файл к поиску или требуется проверка результата. Во время OCR переключение заблокировано, чтобы результат одного файла не попал в другой.

Дополнительные инструменты обрезки, качества и кисти по умолчанию скрыты кнопкой `Показать инструменты`, чтобы рабочая колонка оставалась компактной.

## Запуск

Есть два способа запуска:

1. **Portable-выпуск**: скачайте архив из раздела GitHub Releases, распакуйте и запустите `Запустить приложение.bat`. Python и интернет не требуются.
2. **Исходники из GitHub**: скачайте ZIP нужной ветки и дважды нажмите `run.bat`. Скрипт найдёт Python 3.10–3.12, а при его отсутствии попробует установить Python 3.12 через Windows `winget`. Затем он создаст `.venv` и установит зависимости. Для первого запуска нужен интернет.

Для обычной работы рекомендуется portable-выпуск: в нём уже есть Python, PyTorch, OCR и модели YOLO.

[Скачать portable v4.20](https://github.com/nikolasjirovski-collab/parts-hotspot-v4-fragments/releases/tag/v4.20-portable)

## Правильный порядок работы

1. Нажмите `Добавить файлы` и выберите один или несколько PDF либо изображений.
2. Для PDF выберите нужную страницу кнопками `← PDF` и `PDF →`.
3. Если нужна только часть схемы, включите `Выделять область мышью`, выделите область и нажмите `Применить`.
4. Вставьте список номеров и артикулов.
5. Нажмите `Найти по списку`. Эта кнопка всегда очищает старую расстановку и запускает полный поиск заново.
6. Дождитесь статуса `Объединённый OCR` или другого итогового источника распознавания.
7. Проверьте точки вручную: их можно перетаскивать мышью, удалять и добавлять кликом.
8. Нажмите `Экспорт CSV` или `Экспорт JSON`.

Если результат плохой, нажмите `Очистить точки`, поправьте обрезку или список и снова нажмите `Найти по списку`. Отдельного запуска OCR без списка больше нет: поиск всегда фильтруется по номерам из списка, иначе мелкие схемы дают нестабильные случайные совпадения.

Каждая открытая картинка и каждая текущая страница PDF приводится к рабочему формату `800x800`. Если применить обрезку, результат обрезки тоже сразу переводится в `800x800`.

Вставка списка артикулов работает через `Ctrl+V` и на русской раскладке клавиатуры (`Ctrl+М`).

Картинку можно приближать и отдалять колесиком мыши. Чтобы сдвинуть увеличенную область, зажмите левую кнопку мыши на пустом месте картинки и потяните.

## Формат артикулов

В каждой строке первым должен идти номер с PDF, после него артикул.

```text
1 ABC-123
2 456789
№3 - ART-77
4; ZX-9000
PG8710I-5 опора ручки
```

Приложение связывает номер `1`, `2`, `3`, `4`, `5` с соответствующим артикулом.
Если строка начинается с артикула вида `PG8710I-5`, позиция берется из суффикса после дефиса.

Номера сравниваются без ведущих нулей: `01` и `1` считаются одним номером.

## PDF

PDF открывается постранично. Кнопки `← PDF` и `PDF →` переключают страницы. OCR и ручная разметка работают с текущей страницей.

Если в PDF есть текстовый слой, приложение сначала извлекает цифры напрямую из PDF вместе с координатами. Это не OCR и обычно точнее для векторных каталогов.

Если текстового слоя нет, приложение использует каскад распознавания по изображению страницы:

1. 800 grid OCR - несколько перекрывающихся проходов по картинке `800x800`: `256x256` с шагом `128`, `320x320` с шагом `160`, `400x400` с шагом `200` и полный кадр `800x800`.
2. Windows OCR - встроенный OCR Windows.
3. Line label OCR / detector - поиск подписей возле коротких горизонтальных линий-выносок.
4. Circled number detector - локальный детектор номеров в кружках.
5. OpenCV digit detector - локальный детектор печатных цифр на схемах.
6. OpenCV multiscale detector - повторный поиск маленьких цифр на увеличенных копиях изображения.
7. Tesseract - если Tesseract установлен отдельно и добавлен в `PATH`.
8. YOLO detector - the primary `models/yolo_numbers.pt` model is followed by the SY-adapted `models/yolo_numbers_sy.pt` model while positions from the input list are still missing. Results from both passes are deduplicated.

YOLO settings:

- `PARTS_YOLO_MODEL` - custom `.pt` model path. If omitted, the app uses the installed primary and SY models. Setting this variable intentionally disables the ensemble and uses only the specified model.
- `PARTS_YOLO_IMGSZ` - inference size, default is selected automatically between 1280 and 2200.
- `PARTS_YOLO_CONF` - YOLO confidence threshold, default is `0.18` for number-aware models and `0.25` for generic pretrained models.

Для PDF без нормального текстового слоя приложение отдельно отрисовывает текущую страницу в увеличенном масштабе для OCR. Это не меняет сам PDF и нужно только для поиска мелких номеров на качественных схемах.

Для плотных схем с линиями-выносками приложение дополнительно ищет подписи рядом с короткими горизонтальными линиями и может восстановить короткий пропуск по соседним найденным номерам, например `113`, `114`, `115`. Такие точки можно проверить и подвинуть вручную.

Все найденные координаты переводятся обратно в общую систему `800x800`, одинаковые номера рядом объединяются, затем результат сравнивается со вставленным списком артикулов.

## Быстрая обрезка

1. Включите `Выделять область мышью`.
2. Протяните рамку по картинке или странице PDF.
3. Нажмите `Применить`, если нужно продолжить работу уже с обрезанной картинкой.
4. Нажмите `Сохранить обрезанное`, если нужно сразу сохранить результат.

Для изображений сохраняется новый PNG/JPEG/WebP-файл.

Для PDF сохраняется новый PDF с обрезанной областью. Если включен флажок `PDF: применить ко всем страницам`, такая же относительная область применяется ко всем страницам документа. Если флажок выключить, обрезается только текущая страница.

## Экспорт

`Экспорт CSV` сохраняет таблицу для Excel: номер, артикул, источник, страница PDF, координаты цифры и источник OCR.

`Экспорт JSON` сохраняет техническую разметку: координаты, номера, артикулы и параметры исходного файла.

## OCR

Правильный алгоритм распознавания:

1. Открытая страница или картинка приводится к `800x800`.
2. Если включена обрезка, сначала берется только выделенная область, потом она снова приводится к `800x800`.
3. Приложение делает несколько перекрывающихся проходов: `256x256` с шагом `128`, `320x320` с шагом `160`, `400x400` с шагом `200` и полный кадр `800x800`.
4. Все координаты переводятся обратно в общую систему `800x800`.
5. Одинаковые номера рядом объединяются.
6. Результат сравнивается со списком номеров и артикулов.
7. Пропуски между соседними найденными номерами восстанавливаются как точки `Inferred sequence` или `Inferred long sequence`. Это черновые точки для ручной проверки.
8. Найденные точки добавляются в таблицу для ручной проверки и сохранения.

Если нужен Tesseract, установите отдельно Tesseract OCR для Windows и добавьте `tesseract.exe` в `PATH`. Тогда приложение сможет использовать его как запасной OCR-бэкенд.
